About

Me

Hello, and welcome!

I'm Sam Liman, a Software Engineer living in Denver, Colorado.

People are frequently surprised to hear that I am a second-generation native to such a colorful state. Until they get to know me. I am drawn towards a myriad of creative endeavors, including photography, cooking, programming, and therapeutic massage, to name a few.

This diversity is reflected in my musical tastes, as well. I was raised on bands like The Beatles, Pearl Jam, and Metallica; and, though today I primarily listen styles of electronic music that don't conform with most common definitions, my preferences remain quite diverse. For example, the dark and atmospheric melodies of Peverelist could be described as UK Garage and Techno. Jacques Greene's work loosely fits the definitions for House Music, but he primarily creates using analog synthesizers; blending the results with jazz-like dynamism. Meanwhile, Christian LÃ¶ffler produces dancehall-friendly, melodic House Music, frequently combining acoustic instruments with his own production.

I began working with computers in 2015 as an 'IT Guy', moving back to my hometown, Steamboat Springs, Colorado, to pursue this opportunity. After 4 years in the field, I felt stuck, and that I needed to expand my skill set more substantially than I was able to by working in positions which were based on waiting for things to break. This is what compelled me to learn software development. To move to what feels like "the other side" of the computer.

The skills I shaped while working in IT have proven invaluable to me as a software engineer, and I am excited for the opportunity to exercise them within my creativity as a software developer.

Portfolio

Check out what I've made so far

Beatless

HTML5, Javascript (ES6), NodeJS, ExpressJS, EJS, CSS, and MongoDB. Deployed using Heroku. 1 week.


This was the first web application I ever built. Beatless was a very simple music sharing application, allowing a user log in to a specific profile and upload links to songs to share with others. If there is a link to an image shared as well, the image will have that link attached to it on the share page.

This project demonstrated to me just how essential real-time, responsive frameworks like React, Vue, ~and Angular have become to the way we use the internet today.

Carpe Diem Weather

HTML5, React, Python, Flask, Open Weather Map API, Javascript (ES6), NodeJS, ExpressJS, CSS, Postman, Wireframing, User Stories. Deployed using Heroku. 2 weeks.


This was A team project utilizing paired-coding and daily stand-ups to assess our progress. We created a fully authenticated React app that used a Flask database to manage which data to gather from the Open Weather Map API.
My Teammates were Matt Welbourn (https://www.linkedin.com/in/matt-welbourn-58ba6457) and Yettsy Knapp (https://www.linkedin.com/in/yettsy-jo-knapp) GitHub Repositories: Flask: https://github.com/mlwelbourn/Weather-Flask-App.git React: https://github.com/yettsyjk/WeatherReactApp.git

Spotless

HTML5, React, Python, Javascript (ES6), NodeJS, ExpressJS, CSS, and MongoDB. Deployed using Heroku. 2 weeks.


My goal for this app was to create a basic radio app which utilized machine learning to offer to offer users new tracks that they would like. Due to extremely limited time constraints, goal was tabled, and what you see here is a simple playlist management app. Similar to Beatless on the surface, this is a functional React App. This project will continue to evolve .
